# Financial accounts, transfers, and attribution are separate ledger concepts

Fidy will introduce first-class **Financial Accounts** with account kinds such as checking, savings, wallet, cash, and credit card. **Transactions** remain expense or income records that belong to one financial account, while **Transfers** become a separate atomic record for value moving between two explicit sides and must not count as spending, income, or budget consumption. Account attribution is modeled explicitly with `confirmed`, `inferred`, and `unresolved` states, unresolved items go to a review queue, credit-card billing profile data is optional and user-configured, one-sided transfer inference is deferred in v1 because the available capture evidence is too broad to classify safely, and a default financial account is auto-created before onboarding sync so capture can start working on day zero.
